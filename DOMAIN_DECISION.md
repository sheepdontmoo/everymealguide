# Every Meal Guide Domain Decision

## Recommendation

Use a clear, comparison-led `.com` if available. The domain should immediately tell normal people what the site does.

Best naming direction:

```text
[dinner/meal] + [compare/deals/guide]
```

## Top Domain Concepts To Check

1. `everymealguide.com`
2. `comparemeal.com`
3. `mealcompare.com`
4. `dinnerdeals.com`
5. `mealdeliverycompare.com`
6. `easydinnerdeals.com`
7. `bestdinnerdeals.com`
8. `mealboxcompare.com`
9. `dinnerkitcompare.com`
10. `preparedmealcompare.com`

## Best Fit If Available

`everymealguide.com`

Why:

- Short.
- Brandable.
- Broad enough for meal kits, prepared meals, deals, reviews, and countries.
- Matches current project name.
- Easier to remember than long exact-match domains.

## Backup Strategy

If `everymealguide.com` is unavailable or expensive:

1. Prefer `mealcompare.com` or `comparemeal.com` if affordable.
2. Use `mealdeliverycompare.com` if you want more exact SEO clarity.
3. Use `easydinnerdeals.com` if the site becomes more deal-led.
4. Avoid hyphens unless every clean option is unavailable.
5. Avoid weird TLDs for an affiliate comparison site unless the `.com` market is impossible.

## Avoid

- Names that sound like a recipe blog only.
- Names that lock the site into one country.
- Names that lock the site into meal kits only.
- Names that imply official brand affiliation.
- Names that overpromise coupons or discounts.

## Buying Checklist

Before buying:

1. Check exact `.com` availability.
2. Check renewal cost, not just first-year price.
3. Search the name for obvious trademark risk.
4. Check whether social handles are reasonably available.
5. Confirm the name works as an email, for example `hello@domain.com`.

## After Buying

1. Set `SITE_URL` to the final domain.
2. Set `CONTACT_EMAIL` to the new mailbox.
3. Run `npm run build`.
4. Deploy.
5. Confirm `/robots.txt` and `/sitemap.xml` use the final domain.
